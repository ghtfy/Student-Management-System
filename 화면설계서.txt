3. 입력 필드 상세 설명
================================================================================

3.1 "이름" 필드
- 컴포넌트: JTextField
- 용도: 학생 이름 입력
- 유효성: 필수 입력 
- 제약: 특수문자 없음 권장

3.2 "학번(PK)" 필드
- 컴포넌트: JTextField
- 용도: 학생 학번 입력 (기본키)
- 유효성: 필수 입력, 중복 불가
- 수정 모드: 읽기 전용 (editable=false)

3.3 "학과" 필드
- 컴포넌트: JTextField
- 용도: 학과 정보 입력
- 유효성: 필수 입력
- 제약: 특수문자 없음 권장

4. 테이블 상세 설명
================================================================================

4.1 테이블 구조
┌────────────────────────────────────────────────────────────┐
│ 열번호 │ 열이름  │ 타입    │ 너비 │ 내용                  │
├────────────────────────────────────────────────────────────┤
│   0    │ "이름"  │ String  │ 100  │ 학생 이름             │
│   1    │ "학번"  │ String  │ 100  │ 학번(PK)             │
│   2    │ "학과"  │ String  │ 200  │ 학과명                │
└────────────────────────────────────────────────────────────┘

4.2 테이블 특성
- 선택 방식: 싱글 로우 선택 (한 행만 선택 가능)
- 편집 불가: 테이블 직접 수정 불가 (버튼으로만 수정)
- 스크롤: 데이터 많을 시 JScrollPane으로 스크롤 가능
- 자동 새로고침: CRUD 작업 후 즉시 새로운 데이터 로드

4.3 행 선택 이벤트
```
MouseListener 연결:
- mouseClicked() 이벤트 발생
- 선택된 행의 데이터를 입력 필드에 자동 채우기
- 학번 필드는 수정 불가로 설정 (setEditable(false))
```

5. 버튼 상세 설명
================================================================================

5.1 "등록 (Create)" 버튼
┌────────────────────────────────────────────────────────────┐
│ 크기: 일반 버튼 크기                                        │
│ 위치: 하단 버튼 그룹 중 첫 번째                             │
│ 기능:                                                      │
│  1. 입력 필드의 이름, 학번, 학과 데이터 수집               │
│  2. 빈 필드 검사 (모두 입력되어야 함)                      │
│  3. INSERT SQL 실행 (학번 중복 체크)                       │
│  4. 성공 메시지 표시 후 필드 초기화                        │
│  5. 테이블 자동 새로고침                                   │
└────────────────────────────────────────────────────────────┘

5.2 "수정 (Update)" 버튼
┌────────────────────────────────────────────────────────────┐
│ 크기: 일반 버튼 크기                                        │
│ 위치: 하단 버튼 그룹 중 두 번째                             │
│ 기능:                                                      │
│  1. 테이블에서 선택된 행의 학번 확인                       │
│  2. 입력 필드의 이름, 학과 데이터 수집                     │
│  3. UPDATE SQL 실행 (학번 WHERE 조건)                      │
│  4. 성공/실패 메시지 표시                                  │
│  5. 테이블 자동 새로고침                                   │
│ 제약: 학번은 WHERE 조건이므로 수정 불가                   │
└────────────────────────────────────────────────────────────┘

5.3 "삭제 (Delete)" 버튼
┌────────────────────────────────────────────────────────────┐
│ 크기: 일반 버튼 크기                                        │
│ 위치: 하단 버튼 그룹 중 세 번째                             │
│ 기능:                                                      │
│  1. 입력 필드의 학번 확인                                  │
│  2. 확인 다이얼로그 표시 (Yes/No)                          │
│  3. "Yes" 선택 시 DELETE SQL 실행                         │
│  4. 성공 메시지 표시                                       │
│  5. 테이블 자동 새로고침, 필드 초기화                      │
│ 안전성: 삭제 전 사용자 확인 필수                           │
└────────────────────────────────────────────────────────────┘

5.4 "비우기" 버튼
┌────────────────────────────────────────────────────────────┐
│ 크기: 일반 버튼 크기                                        │
│ 위치: 하단 버튼 그룹 중 네 번째                             │
│ 기능:                                                      │
│  1. 모든 입력 필드 초기화 (빈 문자열 설정)                 │
│  2. 학번 필드를 수정 가능하도록 설정                       │
│  3. 테이블 선택 해제 (clearSelection())                    │
│ 용도: 새로운 입력을 시작하거나 필드 리셋                   │
└────────────────────────────────────────────────────────────┘

6. 사용자 상호작용 플로우
================================================================================

6.1 프로그램 시작
1. MainApp 실행
2. 생성자에서 UI 초기화
3. readStudents() 호출 → 모든 학생 정보 로드
4. 테이블에 학생 6명 표시

6.2 새 학생 등록 (CREATE)
사용자 입력 → [이름, 학번, 학과] 
    ↓
[등록 버튼] 클릭
    ↓
입력값 유효성 검사
    ↓
DB INSERT 실행
    ↓
성공/실패 메시지 → 테이블 새로고침

6.3 학생 정보 수정 (UPDATE)
테이블 행 선택
    ↓
입력 필드에 자동 완성 (학번 읽기전용)
    ↓
이름/학과 정보 수정
    ↓
[수정 버튼] 클릭
    ↓
DB UPDATE 실행
    ↓
성공/실패 메시지 → 테이블 새로고침

6.4 학생 정보 삭제 (DELETE)
테이블 행 선택
    ↓
[삭제 버튼] 클릭
    ↓
확인 다이얼로그 표시
    ↓
[Yes] 선택
    ↓
DB DELETE 실행
    ↓
성공 메시지 → 테이블 새로고침

7. 반응성 및 성능
================================================================================

7.1 UI 반응성
- 버튼 클릭: 즉시 반응 (밀리초 단위)
- 테이블 업데이트: 1초 이내 (DB 쿼리 시간 제외)
- 입력 필드: 실시간 입력 (지연 없음)

7.2 성능 최적화
- 연결 풀링: 각 작업마다 새로운 연결 사용
- 쿼리 최적화: PreparedStatement 사용
- 메모리 관리: 이벤트 리스너 적절한 해제
